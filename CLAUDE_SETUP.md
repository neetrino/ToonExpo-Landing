# Настройка Claude Code — Что нужно изучить

## Что настраивается 1 раз (глобально)
- Авторизация через подписку claude.ai
- `~/.claude/settings.json` — разрешения и общие настройки для ВСЕХ проектов

## Что настраивается для каждого проекта отдельно
- `CLAUDE.md` — инструкции для конкретного проекта (стек, правила, команды)
- `.claude/settings.json` внутри папки проекта — если нужны особые настройки только для этого проекта

---


## 1. Авторизация (Подписка Claude.ai)
- Ты используешь Claude Code через подписку Claude Pro/Max — API ключ не нужен
- Авторизация происходит автоматически через аккаунт claude.ai
- Если Claude Code просит войти: выполни `claude` в терминале и следуй инструкции
- Проверить статус: `claude --version`

## 2. Глобальные настройки (~/.claude/settings.json)
Сейчас файл пустой `{}`. Полезные настройки:
```json
{
  "permissions": {
    "allow": [
      "Bash(npm:*)",
      "Bash(git:*)"
    ]
  }
}
```
- `allow` — команды, которые Claude может выполнять без подтверждения
- Документация: https://docs.anthropic.com/en/docs/claude-code/settings

## 3. CLAUDE.md — Инструкции для проекта
Создай файл `CLAUDE.md` в корне проекта. В нём пишешь:
- Что за проект (стек, цели)
- Правила кодирования
- Что нельзя трогать
- Какие команды использовать для запуска/тестирования

Пример:
```markdown
# ToonExpo Landing

## Стек
- Next.js, Tailwind CSS
- Деплой: Vercel

## Команды
- `npm run dev` — локальный сервер
- `npm run build` — сборка

## Правила
- Не менять файлы в /public/legacy/
- Использовать только TypeScript
```

## 4. MCP Серверы (дополнительные инструменты)
У тебя уже установлены плагины (GitHub, Supabase, Linear и др.).
Чтобы активировать — нужно настроить `.mcp.json` или добавить в settings.json.
Изучи: https://docs.anthropic.com/en/docs/claude-code/mcp

## 5. Разрешения (Permissions)
Claude спрашивает подтверждение перед опасными действиями.
Режимы работы:
- **Default** — спрашивает разрешение на bash команды
- **Auto-approve** — через настройки `allow` в settings.json

## 6. Память (Memory)
Claude сохраняет заметки о проекте в папке:
`~/.claude/projects/.../memory/`

Это помогает сохранять контекст между сессиями.
