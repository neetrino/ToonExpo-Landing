# Մեդիա պանակներ, CSV և լենդինգ — պլան և կարգավիճակ

## 1. Նպատակ

- Յուրաքանչյուր նախագիծ ունի **`mediaFolderId`** (CSV-ում «Project ID») — այն համընկնում է `public/project/{mediaFolderId}/` պանակի անվան հետ։
- Ֆայլերը բաշխվում են ենթապանակներով՝ **`Exterior`**, **`Interior`**, **`3DFloorplan`**, **`2Dfloorplan`**։
- Լենդինգում **նախապես** օգտագործվում են ֆայլային ուղիները, եթե ֆայլ գոյություն ունի։ Հակառակ դեպքում — նախկին **DB/Excel URL-ները** (`expo_field_43` …)։

## 2. CSV `ToonExpoData2026.csv`

- Բաժանարար `;`, UTF-8։
- **Նոր ձևաչափ.** Առաջին տողում երրորդ սյունակում կա **«Project ID»** — այդ դեպքում սյունակների քարտեզը հետևյալն է․
  - սյուն 1 → `expo_field_01`
  - սյուն 2 → լրացուցիչ (չի պահվում DB-ում որպես expo)
  - սյուն 3 → **Project ID** → պահվում է `Project.mediaFolderId`
  - սյուն 4+ → `expo_field_02` … `expo_field_53` (52 դաշտ)
- **Հին ձևաչափ** (առանց Project ID սյունակի) — 53 սյունակ `expo_field_01` … `expo_field_53`, `mediaFolderId` = null։

## 3. Ֆայլային կանոններ (անուններ)

| Տեղ | Պանակ | Ֆայլ | Նշում |
|-----|--------|------|--------|
| Hero | `Exterior` | բազային անուն **`1`** | `1.jpg` / `1.png` / `1.webp` … |
| About — մեծ | `Exterior` | **`2`** | + `absolute … bg-black/18` (կարգավորում է UI) |
| About — փոքր | `Interior` | **`1`** | Եթե չկա — չցուցադրել |
| Գալերեա | բոլորը հերթականությամբ | բոլոր պատկերները | `Exterior` ամբողջությամբ → `Interior` → `3DFloorplan` → `2Dfloorplan` |
| Infrastructure | `Exterior` | **`2`** և **`3`** | Նույն «2»-ը կարող է համընկնել About մեծ պատկերի հետ |

Թույլատրելի ընդլայնումներ՝ `jpg`, `jpeg`, `png`, `webp`, `gif` (անունը case-insensitive)։

## 4. Excel դաշտեր (հղումներ, ոչ պատկերներ)

- **43․ Տիպային ինտերակտիվ տուր** → կոդում `expo_field_45`
- **44․ Տեսահոլովակ** → `expo_field_46`

Լենդինգի «Tours & Media» բլոկում ցուցադրվում են այս երկու հղումները (ըստ լրացվածության)։ Արտաքին ինտերակտիվ տուրը (`expo_field_47`) կարող է մնալ պահուստային/հետագայում առանձին որոշման։

## 5. UI փոփոխություններ

- **Gallery** — մեկ ընդհանուր բլոկ, սլայդեր/սլաքներ, պատկերների հերթականություն՝ §3։ Պիտակներ՝ ընդհանուր «Gallery» (ոչ «Typical Renders» …)։
- About / Infrastructure — դասավորություն պահել ըստ Figma, աղբյուրը՝ §3։

## 6. Տեխնիկական

- `Project.mediaFolderId` — Prisma։
- `resolveProjectFolderMedia(mediaFolderId)` — server-only, `fs` + `public/project/...`։
- Լենդինգի `page.tsx` — բեռնում է `mediaFolderId` և փոխանցում լուծված URL-ները։
